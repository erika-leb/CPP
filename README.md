# C++ Modules (42)

A series of projects designed to explore and master core concepts of C++ (C++98 standard), built as part of the 42 curriculum.  
The objective is to move from procedural programming in C to robust, modular, and object-oriented software design.

---

## 🚀 Features

- Object-Oriented Programming (OOP)
- Encapsulation, inheritance, polymorphism
- Operator overloading
- Canonical class form (rule of three)
- Exception handling
- Templates (generic programming)
- Abstract classes and interfaces
- Memory management and resource ownership

---

## 🧠 What I learned

This project series focuses on writing clean, maintainable, and safe C++ code:

- Designing classes with clear responsibilities
- Managing resources manually (constructors, destructors, copy semantics)
- Understanding deep vs shallow copy
- Using polymorphism through virtual functions
- Implementing and using templates
- Handling errors with exceptions
- Writing modular and reusable code

---

## ⚙️ Structure

The project is divided into multiple modules, each targeting specific concepts:

- **Module 00–01**: Basics, classes, member functions  
- **Module 02–04**: Canonical form, operator overloading, inheritance  
- **Module 05–06**: Exceptions, casting  
- **Module 07–09**: Templates, containers, algorithms  

Each module includes several exercises increasing in complexity.

---

## 🛠️ Tech stack

- Language: C++ (C++98 standard)  
- Compilation: `c++ -Wall -Wextra -Werror -std=c++98`  
- Environment: Linux
